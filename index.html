What is PVN ?

A Peers Virtual Network (PVN) is a simply organized but dense network of Internet Users(Peers) who share their unused bandwidths. This Network works like normal VPN which encrypts your internet connection and routes it through a secure server, hiding your online activities and protecting your privacy. You already know in general that with VPN, you can:

  • Browse Anonymously: Keep your online identity and activities private.
  • Access Geo-Restricted Content: Bypass geo-blocks to enjoy your favorite shows and websites from anywhere.
  • Secure Public Wi-Fi: Protect your data from hackers when using public Wi-Fi networks.

Curve25519 in WireGuard

WireGuard is a modern VPN protocol that emphasizes simplicity, speed, and security. One of the key components of its security model is the use of Curve25519 for cryptographic operations. Here's how it fits into WireGuard:

Key Exchange

  • Private and Public Keys: Each WireGuard peer generates a private key and a corresponding public key using Curve25519. The private key is kept secret, while the public key is shared with other peers.
  • ECDH (Elliptic-Curve Diffie-Hellman): WireGuard uses Curve25519 for the ECDH key exchange. When two peers want to establish a secure connection, they exchange their public keys. Each peer then uses its private key and the other peer's public key to compute a shared secret. This shared secret is used to encrypt and decrypt the data sent between the peers.

Security Benefits

  • Strong Security: Curve25519 provides 128 bits of security, which is robust against current cryptographic attacks.
  • Efficiency: The curve is designed for high performance, allowing WireGuard to establish connections quickly and with minimal computational overhead.
  • Resistance to Side-Channel Attacks: Curve25519 is designed to be resistant to side-channel attacks, which makes it suitable for use in environments where security is critical.

Practical Example

  1. Peer A: Generates a private key (A_private) and derives a public key (A_public).
  2. Peer B: Generates a private key (B_private) and derives a public key (B_public).
  3. Key Exchange: Peer A sends A_public to Peer B, and Peer B sends B_public to Peer A.
  4. Shared Secret:
    • Peer A computes the shared secret using A_private and B_public.
    • Peer B computes the shared secret using B_private and A_public.
  5. Secure Communication: Both peers now have the same shared secret, which they use to encrypt and decrypt their communications.
